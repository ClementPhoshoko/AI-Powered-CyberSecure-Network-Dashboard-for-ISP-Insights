# Nginx Configuration

```nginx
server {
    listen 80;
    server_name _;

    client_max_body_size 500M;

    root /var/www/AI-Powered-CyberSecure-Network-Dashboard-for-ISP-Insights/client/dist;
    index index.html;

    # ---------------------------
    # FRONTEND (React SPA)
    # ---------------------------
    location / {
        try_files $uri $uri/ /index.html;
    }

    # ---------------------------
    # BACKEND API
    # ---------------------------
    location /api/ {
        proxy_pass http://127.0.0.1:5000;

        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # ---------------------------
    # SWAGGER DOCS
    # ---------------------------
    location /api-docs {
        proxy_pass http://127.0.0.1:5000/api-docs/;

        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Test configuration

```bash
sudo nginx -t
```

Restart

```bash
sudo systemctl restart nginx
```

Enable

```bash
sudo systemctl enable nginx
```

# Firewall

Open only

* Port 22 (SSH)
* Port 80 (HTTP)
* Port 443 (HTTPS later)

Do **not** expose port **5000** publicly.

### Critical Oracle Cloud Step (iptables reset):
Oracle Cloud instances often have default iptables rules that block traffic even with ufw. First reset and clear them:

```bash
# Reset iptables rules completely
sudo iptables -F
sudo iptables -X
sudo iptables -Z
sudo iptables -P INPUT ACCEPT
sudo iptables -P OUTPUT ACCEPT
sudo iptables -P FORWARD ACCEPT

# Save iptables rules (if iptables-persistent is installed)
sudo netfilter-persistent save 2>/dev/null || true
```

### Configure ufw:
```bash
sudo ufw reset
sudo ufw allow 22/tcp
sudo ufw allow 80,443/tcp
sudo ufw enable
```

# Automatic Deployment (CI/CD)

Deployment is handled using **GitHub Actions**.

Every push to the `main` branch automatically:

1. Connects to the Oracle Cloud server via SSH.
2. Pulls the latest code.
3. Installs updated dependencies if needed.
4. Builds the React application.
5. Copies the new build to `/var/www/speedtest`.
6. Restarts the Express API using PM2.

This provides a deployment workflow similar to Vercel while keeping full control of the infrastructure.

# Security Notes

* Never commit `.env` files.
* Keep the Supabase Service Key private.
* Keep port **5000** inaccessible from the internet.
* Serve all traffic through Nginx.
* Use PM2 to ensure automatic application recovery.
* Enable HTTPS once a domain is configured.
